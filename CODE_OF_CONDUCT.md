# Code of Conduct

## TL;DR

Be kind. Argue the work, not the person. Skip the cliché AI doomer / boomer
debates in this issue tracker — there are 10,000 other places for those.

## Specifics

- **Stay on topic.** Issues, PRs, and Discussions are about Hydra and the
  supervision pattern. General AI ethics debates, CTF philosophy debates,
  and tool-flame-wars belong elsewhere.
- **No harassment.** Calling code bad is fine. Calling the person who wrote
  it bad is not. Disagree about technical choices; don't make it personal.
- **No CTF flag dumps.** If you've solved a public CTF using Hydra, please
  don't paste real flags in issues or PRs — discuss approach, not output.
- **Responsible disclosure for security issues.** See `SECURITY.md`.
- **No spam, no recruiting in issues, no link drops to unrelated projects.**

## Enforcement

Report problems by opening an issue tagged `code-of-conduct` or, for
sensitive matters, contact the maintainer privately via GitHub.

Maintainer actions, in order of escalation: comment requesting change →
edit + lock conversation → block from repo. We won't tolerate harassment,
but we also won't moderate technical disagreements. Argue, just argue
nicely.
